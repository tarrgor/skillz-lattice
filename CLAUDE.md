# CLAUDE.md

## Skill authoring

All skills created in this directory (`skills/<name>/SKILL.md`) must be brief and precise. Include all information necessary to perform the skill's task correctly, but omit general descriptions, background, or filler that isn't directly related to the skill's specific topic — every sentence should earn its tokens.

## Pull requests

Any PR that resolves a GitHub issue must include a `Closes #<issue-number>` reference in its body, so GitHub closes the issue automatically on merge.
